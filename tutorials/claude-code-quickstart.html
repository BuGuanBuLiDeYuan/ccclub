基本工作流程

1

进入项目目录

cd /path/to/your/project
2

启动Claude Code

claude
3

开始对话编程

使用自然语言描述您想要实现的功能

核心命令

项目管理命令

/init

初始化项目，创建CLAUDE.md配置文件

/init

/clear

清理聊天记录，开始新的会话

/clear

/help

显示所有可用命令和功能

/help

/exit

退出Claude Code会话

/exit

开发工作流命令

/install-github-app

安装GitHub集成，自动审查PR

/install-github-app

/model

切换AI模型（Sonnet 4、Opus 4.1等）

/model sonnet-4

项目初始化

创建 CLAUDE.md 文件

CLAUDE.md 是Claude理解您项目的核心文件，应包含：

CLAUDE.md 模板示例

# 项目名称

## 项目概述
简要描述项目的目的和功能

## 技术栈
- 前端：React/Vue.js
- 后端：Node.js/Python
- 数据库：PostgreSQL/MongoDB
- 其他工具...

## 文件结构
```
src/
  components/    # React组件
  pages/        # 页面文件
  utils/        # 工具函数
  styles/       # 样式文件
```

## 开发规范
- 使用TypeScript进行类型检查
- 遵循ESLint配置
- 组件使用函数式写法
- 命名约定：驼峰式命名

常见任务

代码生成

示例：创建React组件

用户：帮我创建一个用户登录表单组件，包含邮箱和密码字段，以及表单验证
Claude：我来为您创建一个带验证的登录表单组件...

[Claude会自动分析项目结构，创建符合项目规范的组件文件]

代码重构

示例：优化现有代码

用户：重构 src/utils/api.js 文件，添加错误处理和类型检查
Claude：我来分析并重构这个API工具文件...

[Claude会读取文件，理解代码逻辑，然后进行重构优化]

测试和调试

示例：运行测试并修复问题

用户：运行测试套件，如果有失败的测试请帮我修复
Claude：我来运行测试并分析失败原因...

[Claude会执行测试命令，分析错误输出，然后修复代码]

有效的提示技巧

🎯 具体明确

详细描述您想要的功能，包括输入、输出和期望行为

📋 提供上下文

说明相关的文件位置、现有代码结构和项目约束

🔄 迭代优化

根据Claude的输出进一步细化需求和调整实现

🧪 要求测试

明确要求Claude编写测试用例和进行验证

实战示例

示例1：创建新功能

用户指令：

帮我实现一个用户认证系统：
1. 创建用户注册和登录页面
2. 实现JWT token验证
3. 添加密码加密功能

- 确保遵循项目编码规范

示例3：快速修复

用户指令：

修复当前的构建错误，运行 npm run build 
并解决所有类型错误和linting问题

💡 专业提示

使用 /clear 命令经常清理会话，特别是在开始新任务时。这有助于保持上下文相关性并节省token使用。
